Worldline is a generated static headline reader. A scheduled workflow collects public feed metadata, validates and organizes it, and publishes daily JSON files through GitHub Pages. Your browser reads those generated files; it does not contact publisher feeds.

Collection

The workflow combines configured RSS and Atom feeds with Google News RSS editions, date searches, and date-bounded GDELT DOC API requests. Publisher feeds and external APIs are requested by the workflow, not by a visitor's browser.

Each market uses its own time zone, keeps seven local calendar dates, records source health, and protects a valid archive from suspiciously small updates.

Processing

Feed content is treated as untrusted. The pipeline validates URLs and timestamps, removes tracking parameters, strips HTML from summaries, limits response sizes, rejects unsafe schemes, and never executes publisher-provided code or styles.

Duplicates and related coverage

Exact duplicates are identified by normalized URLs and stable publisher identifiers. Near duplicates use conservative headline token and character similarity plus publication-time proximity. Separate articles about the same event may be grouped as related coverage while preserving alternate publisher links.

Ranking and balance

The lead story is not selected only by recency. Configurable signals include independent-source count, recency, source-quality settings, useful source metadata, and domestic relevance. Publisher and provider limits reduce source concentration in the first results.

Automated ranking is a presentation aid, not an objective assessment of importance, quality, or truth.

Translations

Original headlines are always retained. Optional English translations can be created at build time through a configured provider. Machine translations are labeled and may contain errors. The site remains functional without a translation provider.

Status meanings

Data freshness and source coverage can vary by market. Worldline exposes the state of the selected edition and date rather than presenting incomplete coverage as complete.

Current
The daily file was generated successfully from the latest collection.
Partially Updated
Enough new data was produced, but one or more sources failed.
Retained From Previous Update
The latest attempt was unsafe or incomplete, so the last valid daily file remains authoritative.
No Articles Retrieved
No valid articles are available for the selected date.
Data Generation Failed
The latest collection did not produce a usable daily file.
News Not Ready
A daily news file may be unavailable while it is being prepared.
Sample Preview
The date contains clearly labeled repository preview data and has not yet been replaced by a live collection.

Known limitations

RSS availability varies by publisher. Some feeds omit images, descriptions, or precise timestamps. GDELT and Google News may contain aggregator links or incomplete metadata. Event clustering can miss related reporting or, occasionally, group similar stories too broadly. Date-specific historical results depend on what upstream sources expose.
